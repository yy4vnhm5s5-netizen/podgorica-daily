# Project

## Product boundaries

Podgorica Daily is a public information product, not a social network or a generic content aggregator. Public information must remain usable without an account. Accounts are reserved for approved editorial and administrative workflows.

## Delivery standards

- Responsive and accessible behaviour is a baseline, not a later enhancement.
- Every external datum has ownership, provenance, freshness expectations, and failure behaviour.
- Secrets never enter client bundles, repositories, logs, or issue content.
- Changes ship through reviewed pull requests with automated checks and traceable commits.

## Decision records

Material choices—data providers, maps, identity, persistence, and hosting—require an architecture decision record covering cost, licensing, security, privacy, operational ownership, and exit strategy.

## Non-goals for Sprint 0

Sprint 0 does not implement weather, transport, events, maps, search, identity, administration, scraping, or database schemas. It produces the standards and plans that make later work coherent.
